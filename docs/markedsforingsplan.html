Seksjon 1

Produktsammendrag

BiVokter er den ledende norske appen for birøktere — bygget for daglig bruk i bikubene, fra inspeksjonsregistrering og varroa-telling til honninghøstlogg og sesongplanlegging. Appen er tilgjengelig på iOS og Android, og bruker Yr.no for værvarsling, Mapbox for kartvisning og Supabase som sikker skyplattform med norsk datalagring (eu-west-1).

Konfliktpunkter — detaljer

🔴 Dark Mode (#16) — Høy konflikt

constants/colors.ts bruker hardkodede hex-verdier som importeres direkte i alle komponenter. Dark mode krever refaktorering til React Native sin useColorScheme() hook og et dynamisk fargesystem (f.eks. Colors.light.honey vs Colors.dark.honey). Dette er den største enkelt-endringen og vil berøre alle ~40 komponenter i appen.

🔴 AI-rammeanalyse (#6) — Løpende API-kostnad

Krever integrasjon med Claude Vision API (eller tilsvarende). Kostnad per analyse: ~0,01–0,05 USD. Med 1 000 daglige analyser: 300–1 500 USD/mnd i API-kostnader i tillegg til utviklingskost. Bør prises som Pro-/Lag-funksjon med brukskvote.

🟠 Bluetooth-vekt (#10) — Ny native modul

Krever en BLE-kompatibel native modul (f.eks. react-native-ble-plx). Native moduler krever ny EAS-bygg — Expo Go virker ikke. Må testes mot faktiske vektmodeller (Tronix V2, TrueScale). Vurder å starte med manuell inntasting + BLE i v2.

🟠 Anonym benchmark (#8) — RLS-oppdatering

Aggregering på tvers av brukere krever en Supabase Edge Function som kjøres med service_role-nøkkel, ikke anon-nøkkel. RLS-policies på hives og inspections må oppdateres for å tillate anonym aggregering uten å eksponere individuelle rader. Trenger sikkerhetsgjennomgang før deploy.

🔵 expo-widgets (#14) — Beta-API

Expo Widgets er tilgjengelig i Expo SDK 55 men fortsatt i beta. API-overflaten kan endre seg mellom SDK-versjoner. Anbefalt å vente til SDK 56 eller bruke react-native-widgets-extension som alternativ.
