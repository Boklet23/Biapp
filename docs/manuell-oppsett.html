weekly-hive-alerts — Sikkerhetsnøkkel

Beskytter cron-jobben mot uautorisert tilgang

Kritisk — gjør nå
⚠️ Bakgrunn

Uten dette kan hvem som helst som kjenner URL-en utløse mass-push-varsler til alle brukere. Nøkkelen er nå satt opp i koden — du trenger bare å sette de faktiske verdiene.

1
Generer et tilfeldig token

Kjør dette i terminalen (trykk ! foran i Claude Code-terminalen):

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

Du får en lang hex-streng. Kopier den og ta vare på den — du bruker den i steg 2, 3 og 4.

2
Legg inn i Supabase — Edge Function secret

Gå til supabase.com/dashboard → prosjekt zujvhbnuqocquthbujmp
→ Edge Functions → weekly-hive-alerts → Secrets-fanen
→ Add new secret

FeltVerdi
NameWEEKLY_ALERTS_SECRET
Valuetokenet fra steg 1

Klikk Save.

3
Legg inn i Postgres — databasekonfig

Gå til SQL Editor og kjør disse tre kommandoene én etter én:

Erstatt verdiene i < >
-- 1. Alerts-nøkkelen (fra steg 1) ALTER DATABASE postgres SET app.weekly_alerts_secret = 'ditt-token-her'; -- 2. Supabase-URL ALTER DATABASE postgres SET app.supabase_url = 'https://zujvhbnuqocquthbujmp.supabase.co'; -- 3. Service Role Key (Settings → API → service_role) ALTER DATABASE postgres SET app.supabase_service_role_key = 'din-service-role-key';
🔑 Finn service_role key

Supabase Dashboard → Settings → API → Project API keys → service_role (klikk «Reveal»)

4
Kjør migrasjon 0018

Dette oppretter cron-jobben på nytt med x-alerts-secret-headeren:

npx supabase db push

Alternativt: kopier innholdet fra supabase/migrations/0018_weekly_alerts_secret.sql og kjør det direkte i SQL Editor.

5
Verifiser

Test funksjonen manuelt fra Supabase Dashboard → Edge Functions → weekly-hive-alerts → Invoke. Legg til headeren x-alerts-secret: <ditt-token> — du skal få HTTP 200. Uten headeren skal du få HTTP 401.

RevenueCat webhook

Automatisk synk av abonnementsstatus ved kansellering og fornyelse

Kritisk — gjør nå
1
Legg inn REVENUECAT_WEBHOOK_SECRET i Supabase

Gå til Edge Functions → revenuecat-webhook → Secrets
Generer et nytt token (samme metode som oppgave 1, steg 1) og legg det inn:

FeltVerdi
NameREVENUECAT_WEBHOOK_SECRET
Valuenytt tilfeldig token
2
Konfigurer webhook i RevenueCat Dashboard

Gå til app.revenuecat.com → ditt prosjekt → Integrations → Webhooks → Add webhook

FeltVerdi
Webhook URLhttps://zujvhbnuqocquthbujmp.supabase.co/functions/v1/revenuecat-webhook
Authorization headerBearer <REVENUECAT_WEBHOOK_SECRET>

Aktiver disse events: INITIAL_PURCHASE, RENEWAL, CANCELLATION, EXPIRATION, BILLING_ISSUE

3
Test webhook

Klikk Test-knappen i RevenueCat. Sjekk Supabase Dashboard → Edge Functions → Logs og bekreft at du ser en vellykket request.

Google Play — AAB og abonnementsprodukter

Last opp første build og opprett kjøpbare produkter

Høy prioritet
1
Bygg AAB (hvis ingen preview-build er klar)

Preview-profilen er nå konfigurert for AAB-format:

npx eas build --profile preview --platform android --non-interactive

Last ned .aab-filen fra EAS Dashboard når bygget er ferdig.

2
Last opp AAB til Play Console

Gå til play.google.com/console → velg appen no.biapp.app
→ Testing → Internal testing → Create new release
→ last opp .aab-filen → fyll inn release notes → Save → Review → Start rollout

⚠️ Første opplasting

Google Play krever minst én godkjent AAB-opplasting for å kunne opprette In-App Products. Du trenger ikke gå til produksjon — Internal testing er nok.

3
Opprett abonnementsprodukter

Venstre meny → Monetize → Products → Subscriptions → Create subscription

Product IDNavnPrisPeriode
hobbyist_monthlyHobbyist — månedligNOK 491 måned
hobbyist_annualHobbyist — årligNOK 3991 år
profesjonell_monthlyProfesjonell — månedligNOK 1491 måned
profesjonell_annualProfesjonell — årligNOK 1 1901 år
lag_monthlyLag — månedligNOK 4991 måned
lag_annualLag — årligNOK 3 9901 år

Sett hvert produkt til status Active.

4
Konfigurer produktene i RevenueCat

Gå til app.revenuecat.com → Products → Google Play
→ importer produktene (eller legg til manuelt med samme Product ID-er)

Deretter under Entitlements:

Entitlement IDProdukter som hører til

profesjonellprofesjonell_monthly, profesjonell_annual
laglag_monthly, lag_annual

Under Offerings: opprett ett offering og legg til packages.

App-ikon og splash screen

Påkrevd for produksjonsopplasting til Google Play

Medium prioritet
1
Lag ikonene

Bruk Figma, Canva eller Adobe Express. Eksporter som PNG.

FilStørrelseFormat
assets/icon.png1024 × 1024 pxPNG, ingen transparens
assets/android-icon-foreground.png1024 × 1024 pxPNG med transparens — bare forgrunnsmotivet
assets/splash-icon.png1284 × 2778 pxPNG
ℹ️ Android adaptive icon

Bakgrunnsfargen er allerede satt til #1A1A2E i app.json. Foreground-filen skal bare inneholde selve logo-motivet uten bakgrunn.

2
Erstatt filene i assets/-mappen

Plasser de nye filene i assets/-mappen og overskriv de eksisterende.

3
Commit og push
git add assets/icon.png assets/android-icon-foreground.png assets/splash-icon.png git commit -m "feat: app-ikon og splash screen" git push origin master
