RevenueCat Dashboard — 3 manuelle steg

API-et støtter ikke disse operasjonene — må gjøres i nettleseren

Kritisk — gjør nå
✅ Allerede gjort via API

REVENUECAT_WEBHOOK_SECRET er satt i Supabase. 3 entitlements (hobbyist, profesjonell, lag) er opprettet med produkter tilknyttet. Default offering med 6 pakker er opprettet.

⚠️ Kritisk feil: Feil pakkenavn i RevenueCat

RevenueCat-appen er registrert med pakkenavn no.Biapp i stedet for no.biapp.app. Alle kjøp vil feile validering mot Play Store. Dette må rettes manuelt — API returnerer 405 på PATCH/PUT.

1
Fiks pakkenavn: no.Biapp → no.biapp.app

Gå til app.revenuecat.com → ditt prosjekt → Apps
→ klikk på Biapp (Play Store)
→ endre Package Name fra no.Biapp til no.biapp.app
→ klikk Save changes

🚫 Uten denne endringen vil alle kjøp feile

Play Store sender no.biapp.app i alle kjøpsvalideringer. RevenueCat med feil pakkenavn vil avvise dem.

2
Koble produkter til pakker i default offering

Gå til app.revenuecat.com → Offerings → default
→ klikk på hver pakke nedenfor → velg tilhørende produkt under Products:

2

Google Service Account

Én gang — låser opp automatisk EAS submit og abonnements-scriptet

Kritisk — gjør én gang
🤖 Hva dette låser opp

Når dette er gjort kan du kjøre:

  • npx eas build --profile production --auto-submit --non-interactive — bygg + opplasting i én kommando
  • node scripts/create-play-subscriptions.js — oppretter alle 6 abonnementsprodukter automatisk
  • EAS Workflow tar over fremtidige opplastinger automatisk ved push til master
1
Opprett service account i Google Cloud Console

Følg Expo sin guide steg for steg:
https://github.com/expo/fyi/blob/main/creating-google-service-account.md

Kort oppsummert:

StegHandling
1Gå til console.cloud.google.com → opprett eller velg et prosjekt
2Søk etter «Google Play Android Developer API» → Enable
3IAM & Admin → Service Accounts → Create service account
4Gi den et navn, f.eks. biapp-play-submit
5Under Keys → Add key → JSON → last ned filen
6Lagre filen som google-play-service-account.json i prosjektets rotmappe
✅ Filen er allerede i .gitignore

Den vil aldri bli committet til GitHub ved et uhell.

2
Link service account til Google Play Console

Gå til play.google.com/console → Setup → API access
→ velg det Google Cloud-prosjektet du brukte ovenfor → Grant access
→ finn service account-eposten → gi rollen Release Manager
→ klikk Apply

⏱️ Forsinkelse

Det kan ta opptil 24 timer før tilgangen er aktiv første gang. I praksis er det som regel under en time.

3

Første AAB-opplasting til Play Store

Google krever én manuell opplasting før API og abonnementsprodukter fungerer

Høy — kun én gang
⚠️ Google-krav

Google Play API blokkerer oppretting av abonnementsprodukter og automatisk submit inntil én AAB er manuelt godkjent. Etter dette tar EAS over.

1
Bygg en preview-AAB (hvis ingen klar)
npx eas build --profile preview --platform android --non-interactive

Last ned .aab-filen fra EAS Dashboard når bygget er ferdig (~10 min).

2
Last opp til intern testing

Gå til play.google.com/console → velg appen no.biapp.app
→ Testing → Internal testing → Create new release
→ last opp .aab-filen → fyll inn release notes → Save → Review release → Start rollout to Internal testing

3
Vent på godkjenning

Intern testing godkjennes vanligvis innen noen timer. Du vil få e-post fra Google Play.
