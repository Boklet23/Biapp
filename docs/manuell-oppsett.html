RevenueCat webhook

Synkroniserer abonnementsstatus ved kansellering, fornyelse og utløp

Kritisk — gjør nå
⚠️ Ingen REST API

RevenueCat tilbyr ingen API for webhook-konfigurasjon — dette må gjøres i dashboardet. Bekreftet av RevenueCat support.

1
Legg inn REVENUECAT_WEBHOOK_SECRET i Supabase

Generer et nytt tilfeldig token:

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

Gå til supabase.com/dashboard → prosjekt zujvhbnuqocquthbujmp
→ Edge Functions → revenuecat-webhook → Secrets → Add new secret

FeltVerdi
NameREVENUECAT_WEBHOOK_SECRET
Valuetokenet du genererte ovenfor

Klikk Save. Ta vare på tokenet — du bruker det i neste steg.

2
Konfigurer webhook i RevenueCat Dashboard

Gå til app.revenuecat.com → ditt prosjekt → Integrations → Webhooks → Add webhook

FeltVerdi
Webhook URLhttps://zujvhbnuqocquthbujmp.supabase.co/functions/v1/revenuecat-webhook
Authorization headerBearer <REVENUECAT_WEBHOOK_SECRET>

Aktiver disse events: INITIAL_PURCHASE, RENEWAL, CANCELLATION, EXPIRATION, BILLING_ISSUE

3
Test webhook

Klikk Test-knappen i RevenueCat. Gå til Supabase Dashboard → Edge Functions → Logs og bekreft at du ser HTTP 200.

2

Google Service Account

Én gang — låser opp automatisk EAS submit og abonnements-scriptet

Kritisk — gjør én gang
🤖 Hva dette låser opp

Når dette er gjort kan du kjøre:

  • npx eas build --profile production --auto-submit --non-interactive — bygg + opplasting i én kommando
  • node scripts/create-play-subscriptions.js — oppretter alle 6 abonnementsprodukter automatisk
  • EAS Workflow tar over fremtidige opplastinger automatisk ved push til master
1
Opprett service account i Google Cloud Console

Følg Expo sin guide steg for steg:
https://github.com/expo/fyi/blob/main/creating-google-service-account.md

Kort oppsummert:

StegHandling
1Gå til console.cloud.google.com → opprett eller velg et prosjekt
2Søk etter «Google Play Android Developer API» → Enable
3IAM & Admin → Service Accounts → Create service account
4Gi den et navn, f.eks. biapp-play-submit
5Under Keys → Add key → JSON → last ned filen
6Lagre filen som google-play-service-account.json i prosjektets rotmappe
✅ Filen er allerede i .gitignore

Den vil aldri bli committet til GitHub ved et uhell.

2
Link service account til Google Play Console

Gå til play.google.com/console → Setup → API access
→ velg det Google Cloud-prosjektet du brukte ovenfor → Grant access
→ finn service account-eposten → gi rollen Release Manager
→ klikk Apply

⏱️ Forsinkelse

Det kan ta opptil 24 timer før tilgangen er aktiv første gang. I praksis er det som regel under en time.

3

Første AAB-opplasting til Play Store

Google krever én manuell opplasting før API og abonnementsprodukter fungerer

Høy — kun én gang
⚠️ Google-krav

Google Play API blokkerer oppretting av abonnementsprodukter og automatisk submit inntil én AAB er manuelt godkjent. Etter dette tar EAS over.

1
Bygg en preview-AAB (hvis ingen klar)
npx eas build --profile preview --platform android --non-interactive

Last ned .aab-filen fra EAS Dashboard når bygget er ferdig (~10 min).

2
Last opp til intern testing

Gå til play.google.com/console → velg appen no.biapp.app
→ Testing → Internal testing → Create new release
→ last opp .aab-filen → fyll inn release notes → Save → Review release → Start rollout to Internal testing

3
Vent på godkjenning

Intern testing godkjennes vanligvis innen noen timer. Du vil få e-post fra Google Play.

Produksjons-build og opplasting

Siste steg — gjøres kun etter at alt over er på plass

Siste steg
⛔ Ikke bygg produksjon før
  • Splash screen er ferdig (steg 5)
  • RevenueCat webhook fungerer (steg 1)
  • Abonnementsprodukter er aktive i Google Play + RevenueCat (steg 4)

1
Bygg + last opp automatisk (én kommando)

Service account JSON må ligge i prosjektmappen først:

npx eas build --profile production --platform android --auto-submit --non-interactive

2
Fremover: automatisk ved push til master

EAS Workflow-filen (.eas/workflows/build-submit.yml) kjøres automatisk ved neste push — du trenger aldri kjøre manuell build igjen.

🤖 EAS Workflow er aktiv

Se status i expo.dev → ditt prosjekt → Workflows.

3
Verifiser kjøpsflyt

Test med en ekte Google-konto på Internal testing-sporet. Verifiser at tier oppdateres korrekt i Supabase etter kjøp.
