🤖 Halvautomatisert — PowerShell-script er klar

scripts/setup-gcloud-serviceaccount.ps1 gjør steg 1 automatisk. Kjøres i PowerShell etter at du er logget inn i gcloud.

Når dette er gjort låses opp:

  • node scripts/create-play-subscriptions.js — oppretter alle 6 Google Play-produkter
  • npx eas submit --platform android --latest — laster opp AAB til Play Store
  • EAS Workflow tar over fremtidige opplastinger ved push til master
1
Logg inn i gcloud og kjør PS1-scriptet

Åpne PowerShell og kjør disse to kommandoene:

# Interaktiv innlogging (åpner nettleser) gcloud auth login # Scriptet oppretter prosjekt, aktiverer API, service account og laster ned JSON .\scripts\setup-gcloud-serviceaccount.ps1

Scriptet lagrer nøkkelen som google-play-service-account.json i prosjektrotmappen (allerede i .gitignore).

⚠️ gcloud må være installert

Sjekk: gcloud --version. Hvis ikke installert: winget install Google.CloudSDK

2
Link service account til Google Play Console (manuelt — Google-krav)

Gå til play.google.com/console → Setup → API access
→ velg Google Cloud-prosjektet biapp-play-2026
→ finn biapp-play-submit@biapp-play-2026.iam.gserviceaccount.com
→ gi rollen Release Manager → klikk Apply

⏱️ Forsinkelse

Det kan ta opptil en time før tilgangen er aktiv første gang.

✅ Production AAB er allerede bygget

EAS production build ble kjørt 11. mai 2026 og er ferdig. Last ned AAB-en direkte — du trenger ikke bygge på nytt.

⚠️ Google-krav

Google Play API blokkerer oppretting av abonnementsprodukter og automatisk EAS submit inntil én AAB er manuelt lastet opp. Etter dette tar EAS Workflow over alle fremtidige opplastinger.

1
Last ned AAB fra EAS Dashboard

Gå til expo.dev → ditt prosjekt @boklet23/biapp → Builds
→ finn production-bygget fra 11. mai 2026 → last ned .aab-filen.

Alternativt via CLI:

npx eas build:list --platform android --profile production --limit 1

Kopier download-URL og last ned med nettleseren eller curl.

2
Opprett app-oppføring i Play Console (hvis ikke gjort)

Gå til play.google.com/console → Create app
→ Package name: no.biapp.app → App name: BiVokter
→ fyll inn kategori (Lifestyle), innholdsrating, og distribusjonsstatus.

3
Last opp AAB til intern testing

I Play Console → no.biapp.app → Testing → Internal testing
→ Create new release → last opp .aab-filen
→ fyll inn release notes → Save → Review release → Start rollout to Internal testing

4
Vent på godkjenning

Intern testing godkjennes vanligvis innen noen timer. Du får e-post fra Google Play.
