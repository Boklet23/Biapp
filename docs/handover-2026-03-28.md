## Kjente bugs og ГҘpne problemer

### KRITISKE (blokkerer produksjon)

**1. Sentry DSN hardkodet i kildekoden**
- Fil: `app/_layout.tsx:12`
- DSN er hardkodet og havner i git-historikken
- Fix: Flytt til `EXPO_PUBLIC_SENTRY_DSN` miljГёvariabel i `.env.local` og EAS secrets
```typescript
// Endre fra:
dsn: 'https://b48b3d222387e65619940395e2011d33@o4511...',
// Til:
dsn: process.env.EXPO_PUBLIC_SENTRY_DSN,
```

**2. `sendDefaultPii: true` + `mobileReplayIntegration` вҖ” GDPR-risiko**
- Fil: `app/_layout.tsx:16`
- Sender IP, bruker-ID og skjermopptak til Sentry uten samtykke
- Fix: Sett `sendDefaultPii: false`, fjern `mobileReplayIntegration()` inntil personvernvurdering

**3. Schema-mismatch `swarm_reports` вҖ” svirm-rapporten fungerer ikke**
- `services/swarmReport.ts` bruker `status: 'active'` men databasen krever `'open'` eller `'resolved'`
- `contact_info`-kolonnen finnes ikke i migrasjonsfilen `supabase/migrations/0001_initial_schema.sql`
- Fix:
  1. Endre `'active'` вҶ’ `'open'` i `swarmReport.ts` (linje 10 og 38)
  2. KjГёr i Supabase SQL Editor:
  ```sql
  ALTER TABLE swarm_reports ADD COLUMN IF NOT EXISTS contact_info text;
  ```
